# 🦙 Test-Hae-Llama Project Master Plan

## 🎯 Core Objective
로컬 7b 모델(Qwen 등)을 사용하여 복잡한 엔터프라이즈급 자바 프로젝트의 유닛 테스트를 완벽하게 생성하고, 스프링 표준 경로에 자동 저장하며, `gradle build`를 통과시킨다.

## 🛠️ Key Technical Requirements

### 1. 🔬 초세분화 에이전트 (Micro-Manager Agents) - New!
7b 모델의 한계를 극복하기 위해, 하나의 테스트 메서드를 작성하는 과정을 3개의 독립적인 전문 에이전트로 분리한다.
*   **The Mocker (Given Agent):** 오직 Mockito의 `when(...)` 구문 생성에만 집중한다.
*   **The Executor (When Agent):** 테스트 대상 메서드를 호출하는 코드 한 줄만 생성한다.
*   **The Verifier (Then Agent):** AssertJ의 `assertThat(...)` 구문 생성에만 집중한다.
*   **Prompt Strategy:** 각 에이전트에게는 해당 역할에 맞는 **Few-Shot 예제**를 제공하여 "복사 후 채워넣기" 수준으로 난이도를 낮춘다.

### 2. 🚫 정규식 의존 최소화 & 구조적 빌더 (Builder Pattern)
*   복잡한 정규식으로 코드를 파싱하거나 수정하는 것을 지양한다.
*   대신, 파이썬의 `JavaClassBuilder`가 클래스 뼈대(패키지, 임포트, 필드)를 완벽하게 잡고, LLM은 그 안에 들어갈 **"실행 문장(Statement)"**만 채워넣게 한다.

### 3. 🧪 공격적 버그 사냥 (Bug Hunter Mindset)
*   **Comprehensive Negative Testing:** Null, Empty, Boundary, Exception 흐름을 집요하게 파고든다.
*   **Parameterized Testing:** `@ParameterizedTest`와 `@CsvSource`를 활용해 다양한 실패 데이터를 우아하게 처리한다.

### 4. 💎 클린 코드 및 표준 준수
*   **Java 15+ Text Blocks:** JSON 및 멀티라인 문자열에 `"""`를 강제하여 이스케이프 실수를 방지한다.
*   **Spring Path Intelligence:** `src/test/java` 하위의 정확한 패키지 구조에 자동 저장하며 멀티 모듈을 지원한다.
