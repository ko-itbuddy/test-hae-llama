# 🦙 Test-Hae-Llama Project Master Plan

## 🎯 Core Objective
로컬 7b 모델(Qwen 등)을 사용하여 복잡한 엔터프라이즈급 자바 프로젝트의 유닛 테스트를 완벽하게 생성하고, 스프링 표준 경로에 자동 저장하는 "저사양 친화적" AI 엔진을 구축한다.

## 🛠️ Key Technical Requirements

### 1. 🎯 전수 단위 정복 (Total Unit Coverage)
- **Method & Constructor Focus:** 소스 코드의 모든 생성자와 공개 메서드를 물리적으로 리스트업하고, 각 단위별로 독립적인 테스트 세트를 생성한다.
- **Minimum Quote:** 각 단위당 **최소 1개의 성공 케이스**와 **발견 가능한 모든 실패 케이스**를 반드시 생성하도록 강제한다.
- **No Discard Policy:** 출력이 부실하다고 버리지 않는다. 유효한 자바 코드가 나올 때까지 지시사항을 다듬어 재시도(Retry) 한다.

### 2. 🧩 핀셋 문맥 최적화 (Precision Context Slicing)

### 3. 🧪 공격적 버그 사냥 (Bug Hunter Mindset)
- **Comprehensive Negative Testing:** Null, Empty, Boundary, Exception 흐름을 집요하게 파고든다.
- **Parameterized Testing:** `@ParameterizedTest`와 `@CsvSource`를 활용해 다양한 실패 데이터를 우아하게 처리한다.

### 4. 💎 클린 코드 및 표준 준수
- **Java 15+ Text Blocks:** JSON 및 멀티라인 문자열에 `"""`를 강제하여 이스케이프 실수를 방지한다.
- **Spring Path Intelligence:** `src/test/java` 하위의 정확한 패키지 구조에 자동 저장하며 멀티 모듈을 지원한다.
- **Mandatory Disclaimer:** 모든 클래스 상단에 "🚨 이 테스트 코드는 반드시 통과되어야 합니다라마!" 주석을 명시한다.

## 🚨 Current Status
- v0.4.1 기준: 전수 단위 정복(Atomic Coverage) 및 끈질긴 재시도 로직 안착.
- **품질 진단:** 7b 모델의 "잡담"은 억제되었으나, 비즈니스 의도가 담긴 정교한 검증(Assertion)과 완벽한 엣지 케이스 망 구축이 다음 과제임.

## 🚀 30년 차 개발자의 품질 정복 로드맵 (v0.5.0 목표)

### 1. 📜 계약 기반 문맥 주입 (Contract-Driven Context)
- 단순히 코드를 주는 게 아니라, 대상 메서드의 **입력/출력 계약(Contract)**을 사전에 분석하여 "이 값은 반드시 양수여야 함"과 같은 가이드를 프롬프트 최상단에 배치한다.

### 2. 🔢 표준 엣지 케이스 매트릭스 (Edge-Case Matrix)
- Architect에게 감을 맡기지 않는다. 자바 타입 시스템에 따른 **표준 실패 시나리오 매트릭스**를 엔진에 내장하여, 모든 테스트 생성 시 강제로 주입한다.
  - *String:* null, empty, blank, max_length, special_chars
  - *Numeric:* null, 0, negative, max, min, overflow
  - *Collection:* null, empty, large_size, concurrent_mod

### 3. 🎯 비즈니스 의미론적 검증 (Semantic Assertions)
- `assertThat(result).isNotNull()` 수준을 넘어, 메서드 이름과 로직에서 추출한 **비즈니스 목표**를 검증하도록 유도한다. (예: `isDiscounted()`, `hasAuditLogs()`)
